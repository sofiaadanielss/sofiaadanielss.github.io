Markdown

Markdown es un lenguaje de marcado ligero creado por John Gruber que permite formatear texto de manera sencilla usando sintaxis simple. Se convierte a HTML válido y es ampliamente usado en documentación, foros y plataformas como GitHub.

Sintaxis básica para dar formato

Encabezados:
Usa # para crear títulos. El número de # indica el nivel del título.

Título de nivel 1

Título de nivel 2

Texto básico:
Para negrita, envuelve el texto con dos asteriscos o guiones bajos (texto negrita o texto negrita).
Para cursiva, envuelve el texto con un asterisco o guión bajo (texto cursiva o texto cursiva).

Listas:
Para listas numeradas, comienza cada línea con un número y un punto (1., 2., etc.).
Para listas con viñetas, usa un asterisco, guión o signo de más (*, -, o +) al inicio de cada línea.

Vínculos:
La sintaxis es: [Texto](enlace)

Bloques de código:
Puedes incluir código en línea con comillas invertidas (`). Para bloques de código más grandes, usa tres comillas invertidas (```).

Tablas:

Columna 1 Columna 2
Dato 1 Dato 2

Citas:

Esta es una cita

Casos de uso

Documentación y notas:
Es ideal para escribir documentación técnica, notas y apuntes de manera rápida y organizada.

Blogs y sitios web:
Muchos servicios como los sistemas de gestión de contenido (CMS) y los foros soportan Markdown para facilitar la creación de publicaciones.

Escritura general:
Puedes usarlo para crear listas de elementos, escribir correos electrónicos, o incluso para redactar libros o capítulos de libros.

Plataformas de desarrollo:
En el mundo de la programación se usa ampliamente en repositorios como GitHub y para la escritura de documentación.

Editores y herramientas:
Es compatible con muchas aplicaciones: Google Docs, Trello, editores de texto, etc.

Uso de Git y GitHub

Git

Git es un sistema de control de versiones que realiza un seguimiento de los cambios en los archivos. Es especialmente útil cuando varias personas trabajan en los mismos archivos.

Flujo de trabajo típico:

  • Crear una rama desde la copia principal.
  • Modificar archivos de manera independiente.
  • Fusionar cambios mediante combinación (merge).
  • Git sigue los cambios y permite mantener la versión más actualizada.

GitHub

GitHub es una plataforma en la nube para almacenar, compartir y colaborar en código.

Permite:

  • Presentar y compartir proyectos.
  • Administrar cambios en el código.
  • Recibir sugerencias y revisiones de colaboradores.
  • Trabajar en equipo mediante ramas y solicitudes de cambio.

¿Cómo funcionan Git y GitHub juntos?

Al cargar archivos, GitHub los guarda como un repositorio de Git. Cada cambio genera un commit, y puedes trabajar localmente sincronizándolo con el repositorio remoto.

Proceso colaborativo:

  • Extraer cambios recientes del repositorio remoto.
  • Insertar tus propios cambios al repositorio remoto.
  • Git combina los cambios inteligentemente.
  • GitHub facilita el proceso mediante Pull Requests.

Crear un repositorio en GitHub (remoto)

  1. Haz clic en "+" → Nuevo repositorio.
  2. Escribe nombre, descripción, visibilidad.
  3. Puedes incluir un README opcional.
  4. Haz clic en "Crear repositorio".

Crear un repositorio local con Git

Para crear un repositorio en tu computadora:

  1. Abrir terminal y navegar a la carpeta del proyecto.
  2. Ejecutar: git init
  3. Añadir repositorio remoto: git remote add origin <URL>
  4. Subir cambios: git add ., git commit -m "", git push -u origin main

¿Qué es Hugo?

Hugo es un generador de sitios estáticos escrito en Go. Permite crear sitios web mediante plantillas y contenido en Markdown.

¿Qué son GitHub Actions?

Es una plataforma de CI/CD que automatiza procesos dentro de GitHub, como despliegues automáticos.

Crear sitio con Hugo

Instalación (Git Bash)

curl -LO https://github.com/gohugoio/hugo/releases/download/v0.111.3/hugo_0.111.3_Linux-64bit.tar.gz
tar -xzf hugo_0.111.3_Linux-64bit.tar.gz
sudo mv hugo /usr/local/bin/

hugo new site mi-sitio
cd mi-sitio

Añadir tema

git init
git submodule add https://github.com/theNewDynamic/gohugo-theme-ananke.git themes/ananke
echo 'theme = "ananke"' >> config.toml

Crear contenido

hugo new posts/mi-primer-post.md

Configuración de GitHub Actions para desplegar Hugo

name: Deploy to GitHub Pages

on:
  push:
    branches: [ main ]

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
        with:
          submodules: true
          
      - name: Setup Hugo
        uses: peaceiris/actions-hugo@v2
        with:
          hugo-version: 'latest'
          
      - name: Build
        run: hugo --minify
        
      - name: Deploy
        uses: peaceiris/actions-gh-pages@v3
        with:
          github_token: ${{ secrets.GITHUB_TOKEN }}
          publish_dir: ./public
